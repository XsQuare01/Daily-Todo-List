# Daily Todo List

React + TypeScript + Electron으로 만든 데스크톱 할 일 앱입니다.

트레이 기반 메인 팝업, 항상 위에 떠 있는 위젯, 태그별 추가 위젯, 데스크톱 작업 창을 함께 제공해서 하루 할 일을 빠르게 확인하고 정리할 수 있습니다.

## 빠른 요약

- **실행 형태**: Electron 데스크톱 앱
- **주요 UI**: 트레이 팝업, 위젯, 태그 위젯, 데스크톱 작업 창
- **데이터 저장 위치**: `%APPDATA%/daily-todo-list/`
- **Windows 설치 파일 생성**: `npm run build:win`
- **자동 실행 정책**: 기본 OFF, 트레이 메뉴에서 직접 ON
- **릴리스 기준**: `package.json` 버전 + Git 태그 + GitHub Release

## 위젯 사용 설명서

Daily Todo는 두 개의 창으로 동작합니다.

- **메인 팝업**: 트레이 아이콘을 클릭하면 뜨는 본체. 날짜 이동, 필터, 상세 편집 등 전체 기능을 사용합니다.
- **위젯**: 화면 위에 항상 떠 있는 작은 창. 오늘 할 일만 빠르게 확인하고, 인라인으로 추가하거나 체크할 수 있습니다.

두 창은 서로 배타적으로 동작합니다. 메인을 열면 위젯이 숨고, 메인을 닫으면 위젯이 다시 나타납니다.

### 1. 창 열기·닫기

| 동작 | 방법 |
|---|---|
| 메인 팝업 열기 | 트레이 아이콘 클릭 또는 `Ctrl+Shift+N` |
| 메인 팝업 닫기 | 팝업의 X 버튼 또는 바깥 클릭 (포커스 잃으면 자동 숨김) |
| 위젯 보이기/숨기기 | `Ctrl+Shift+D` 또는 트레이 메뉴 "위젯 표시" |
| 빠른 할 일 추가 | `Ctrl+Shift+N` — 메인 팝업이 뜨면서 입력창에 포커스됨 |
| 완전 종료 | 트레이 메뉴 "종료" |

### 2. 위젯 이동·크기 조절

- **이동**: 위젯 헤더(날짜 표시 영역)를 드래그하면 원하는 자리에 배치됩니다.
- **가장자리 스냅**: 드래그를 화면 가장자리 24px 안쪽에 놓으면 자석처럼 12px 여백으로 딱 붙습니다. 모서리에 정확히 붙이고 싶을 때 편합니다.
- **크기 조절**: 위젯 오른쪽 아래 모서리를 드래그해서 크기 변경. 최소 220×160, 최대 800×900.
- **위치·크기 기억**: 이동·리사이즈한 값은 자동으로 저장되어 다음 실행 시 같은 자리에 나타납니다. 모니터가 연결 해제되어도 가장 가까운 화면에 맞춰 조정됩니다.

### 3. 위젯 안에서 할 수 있는 것

- **체크박스 클릭**: 오늘 할 일 완료 처리.
- **인라인 입력**: 하단의 "빠른 추가..." 입력창에 제목 입력 후 Enter. `#태그` 문법 지원.
- **한 줄 모드**: 헤더 오른쪽의 아이콘(↙)을 누르면 가장 급한 항목 1개만 보여주는 초소형 모드로 전환. 다시 누르면 원상복구. 상태는 저장됩니다.
- **자동 페이드**: 마우스가 위젯 위에 없을 때는 70% 투명도로 흐려져서 배경에 자연스럽게 녹아듭니다. 호버하면 선명해집니다.

### 4. 태그별 추가 위젯 (멀티 포스트잇)

할 일을 카테고리별로 따로 띄우고 싶을 때 사용합니다.

1. 메인 팝업에서 할 일에 `#업무`, `#개인` 같은 태그를 붙입니다.
2. 트레이 메뉴 → **"태그 위젯 추가"** 서브메뉴에 태그 목록이 자동으로 나타납니다.
3. 원하는 태그를 클릭하면 해당 태그의 미완료 할 일만 보여주는 새 위젯 창이 열립니다.
4. 각 태그 위젯은 독립된 위치·크기를 기억하며, 앱을 다시 켜도 복원됩니다.
5. 태그 위젯에서 항목을 추가하면 **자동으로 해당 태그가 붙습니다**. (원한다면 추가 `#다른태그`도 같이 쓸 수 있음)
6. 닫기는 트레이 메뉴 맨 아래의 "#태그명 닫기" 항목으로. 닫으면 설정에서도 사라집니다.

태그 위젯은 날짜가 아닌 "태그"로 필터링하므로, 마감일이 오늘이 아닌 항목도 전부 보여줍니다.

### 5. 클릭 통과 모드

위젯을 "보기만 하는 오버레이"로 만들고 싶을 때 사용합니다. 코딩이나 문서 작업 중에 흘끗 보기만 할 때 유용합니다.

- 트레이 메뉴 → **"위젯 클릭 통과"** 체크.
- 활성화되면 모든 위젯(메인 + 태그 위젯)이 마우스 이벤트를 뒤쪽 창으로 통과시킵니다. 즉, 위젯을 클릭해도 아래에 있는 브라우저나 에디터가 클릭됩니다.
- **중요**: 이 모드에서는 위젯 자체를 클릭할 수 없으므로, 끄려면 반드시 **트레이 아이콘**을 통해 체크를 해제해야 합니다.
- 설정은 저장되며, 다음 실행 시에도 유지됩니다.

### 6. 기타

- **자동 실행**: 트레이 메뉴 → "시작 시 자동 실행"을 체크하면 윈도우 로그인 시 Daily Todo가 자동으로 실행됩니다.
- **데이터 저장 위치**: `%APPDATA%/daily-todo-list/`
  - `todos.json`: 할 일 데이터
  - `settings.json`: 위젯 위치·크기, 태그 위젯 목록, 클릭 통과 설정

### 단축키 요약

| 키 | 동작 |
|---|---|
| `Ctrl+Shift+N` | 메인 팝업 + 입력 포커스 (어디서든) |
| `Ctrl+Shift+D` | 위젯 보이기/숨기기 (어디서든) |
| `Enter` (입력 중) | 할 일 추가 |

## Windows 배포 가이드

### 1. 설치 파일 만들기

```bash
$ npm run build:win
```

빌드가 끝나면 `dist/`에 설치 파일과 업데이트 메타데이터가 생성됩니다.

- `dist/daily-todo-list-<version>-setup.exe`
- `dist/latest.yml`

### 2. 자동 실행 정책

이 앱은 설치 직후 **자동 실행을 강제로 켜지 않습니다.**

자동 실행이 필요하면 앱 실행 후 트레이 메뉴에서 **"시작 시 자동 실행"** 을 직접 체크해야 합니다. 즉, 기본값은 OFF이고 사용자 선택으로만 ON 됩니다.

### 3. 로컬 확인 항목

설치 파일을 만든 뒤에는 아래 항목을 확인하면 됩니다.

1. 설치 프로그램이 정상 실행되는지
2. 설치 후 앱이 정상 실행되는지
3. 트레이 아이콘이 보이는지
4. **"시작 시 자동 실행"** 이 기본 해제 상태인지
5. 필요 시 자동 실행을 켰을 때 다음 로그인부터 동작하는지

### 4. 버전 관리와 릴리스

릴리스 버전은 `package.json`의 `version`을 기준으로 관리합니다.

배포 시에는 아래 세 가지를 같은 버전으로 맞춥니다.

1. `package.json` 버전
2. Git 태그 (`v1.0.0` 형태)
3. GitHub Release

자세한 운영 절차는 `docs/release/windows-release.md`를 참고하면 됩니다.

### 5. 기타 플랫폼 빌드

프로젝트에는 Electron Builder 설정이 포함되어 있어서 macOS / Linux 빌드도 가능합니다.

### Build

```bash
# For windows
$ npm run build:win

# For macOS
$ npm run build:mac

# For Linux
$ npm run build:linux
```
