# Daily Todo List 개발 로드맵

`IMPLEMENTATION.md`를 기준으로 현재 설계를 실제 개발 순서에 맞게 다시 정리한 문서입니다.  
이번 버전은 단순 회고형 정리가 아니라, **이미 정의된 기능의 구현 순서**와 **앞으로 추가할 기능 계획**이 함께 보이도록 조정했습니다.

기준은 다음과 같습니다.

- **MVP**: 이 기능이 있어야 앱의 핵심 경험이 성립하는가?
- **v1.1 / v1.2**: 매일 쓰기 편하도록 관리 경험을 다듬는가?
- **v2**: 이 앱의 차별점을 강화하는가?
- **v3**: 다음 확장 기능으로 자연스럽게 이어지는가?

---

## 1. MVP

앱의 정체성을 성립시키는 최소 기능입니다.  
**목표: 트레이에서 빠르게 할 일을 추가하고, 오늘 할 일을 보고, 완료 처리할 수 있다.**

### 1-1. 앱 기본 골격

- Electron + React + TypeScript + Tailwind CSS 환경 구성
- `main / preload / renderer` 구조 구성
- `window.api` IPC 브릿지 기본 연결
- 트레이 아이콘 생성
- 팝업 창 / 기본 위젯 창 생성 및 표시 제어

### 1-2. 기본 데이터 모델

우선 최소 필드만으로 시작합니다.

#### Todo

- `id`
- `title`
- `completed`
- `important`
- `createdAt`
- `dueDate?`
- `tags?`
- `priority?`

### 1-3. 데이터 저장

- `todos.json` 읽기/쓰기
- 앱 시작 시 todos 로드
- 수정 시 자동 저장
- 저장 후 모든 창에 최신 상태 반영

### 1-4. 핵심 IPC

- `todos:get`
- `todos:save`
- `todos:updated`
- `window:hide`

### 1-5. 팝업 창 핵심 기능

- 할 일 추가
- 할 일 완료 토글
- 할 일 삭제
- 중요 표시 토글
- 필터:
  - 오늘
  - 전체
  - 완료
- `#태그` 자동 파싱

### 1-6. 위젯 핵심 기능

- 오늘 날짜의 미완료 할 일 표시
- 우선순위 순 정렬
- 간단한 빠른 추가 입력창
- 스크롤 가능한 목록

### 1-7. 기본 UX

- `Ctrl+Shift+N`으로 팝업 열기 + 입력창 포커스
- 트레이 클릭으로 팝업 열기/숨기기
- 팝업과 위젯 간 기본 전환 동작
- 기본 자동 저장 흐름

## 2. v1.1

MVP를 “실제로 매일 쓰기 편한 수준”으로 끌어올리는 첫 번째 단계입니다.  
**목표: 기본 입력 앱에서 벗어나, 날짜/태그/우선순위 중심으로 정리할 수 있다.**

### 2-1. 팝업 뷰 강화

- 날짜 네비게이터
- 오늘 날짜 뱃지 표시
- 태그 필터 탭
- 태그 선택기
- 중요 필터

### 2-2. Todo 기본 메타데이터 확장

- 우선순위 표시
- 우선순위 수정
- 마감일 입력 및 수정
- D-Day 계산 표시
- 태그 수정

### 2-3. 기본 위젯 고도화

- 위젯 위치 저장
- 위젯 크기 저장
- 앱 재실행 시 위치/크기 복원
- 가장자리 스냅
- 클릭 통과 모드 저장 및 적용

### 2-4. 창 동작 정리

- 팝업 창 포커스 아웃 시 자동 숨김
- 팝업/위젯 표시 상태 정리
- `Ctrl+Shift+D` 위젯 표시/숨김

---

## 3. v1.2

기본 관리 기능 위에 상세 편집 경험을 얹는 단계입니다.  
**목표: 항목을 단순히 추가하는 수준을 넘어, 내용을 세부적으로 관리할 수 있다.**

### 3-1. Todo 상세 편집 UI

- 항목 펼침/접힘 UI
- 메모 blur 저장
- 링크 입력 및 수정
- 외부 브라우저 열기

### 3-2. 서브태스크

- 서브태스크 추가
- 서브태스크 완료 토글
- 서브태스크 진행률 표시
- 목록/상세에서 진행 상태 반영

### 3-3. 위젯 상세 화면

- 할 일 클릭 시 상세 화면 진입
- 목록 ↔ 상세 이동
- 태그 표시
- 링크 열기
- 서브태스크 표시 및 추가
- 완료 처리 후 목록 복귀
- 우클릭 컨텍스트 메뉴

### 3-4. 데스크톱 창 기본 정리

- 데스크톱 창 기본 생성 및 숨김 처리
- 닫아도 종료되지 않고 숨김 처리
- `Ctrl+Shift+M` 데스크톱 창 열기

### 3-5. 네트워크 날짜

- KST 네트워크 날짜 조회
- API 2개 순차 시도
- 실패 시 로컬 KST 계산 fallback

## 4. v2

이 앱의 차별점과 고급 사용성을 만드는 단계입니다.  
**목표: 단순 TODO 앱을 넘어서, 항상 떠 있는 작업 허브가 된다.**

### 4-1. 데스크톱 뷰

- `Pending Lane / Important Lane` 2열 레이아웃
- 다중 선택
- 일괄 작업:
  - 완료 처리
  - 중요 표시
  - 중요 해제
  - 삭제
- Focus Session 패널
- 중요 → 미완료 기준 상위 5개 추천

### 4-2. 타이머 / 스톱워치

- Todo별 독립 타이머
- 동시에 하나만 실행
- 다른 항목 시작 시 이전 타이머 자동 일시정지
- 완료 시 자동 정지
- `elapsedMs` 누적 저장
- 타이머 초기화

### 4-3. 드래그 앤 드롭

- `@dnd-kit/core` 기반 정렬
- 동일 리스트 내 순서 변경
- 변경 후 저장 및 동기화

### 4-4. 태그 위젯

- 트레이 메뉴에서 태그 위젯 추가
- 특정 태그만 필터링된 추가 위젯 생성
- 각 태그 위젯 위치/크기 개별 저장
- 태그 위젯 닫기 및 제거
- 태그 위젯에서 추가한 항목에 태그 자동 부여

---

## 5. v3

현재 설계를 기반으로 다음 릴리즈에서 고려할 확장 기능입니다.  
**목표: 장기 사용성과 데이터 관리 편의성을 강화한다.**

### 5-1. 검색

- 제목/메모/태그 기준 검색
- 팝업과 데스크톱 뷰에서 빠른 검색 진입
- 검색 결과에서 즉시 편집 또는 완료 처리

### 5-2. 반복 할 일

- 매일 / 매주 / 평일 반복
- 완료 시 다음 반복 일정 자동 생성
- 일반 due date와 반복 규칙의 충돌 처리 정의

### 5-3. 정렬 옵션

- 우선순위순 / 마감일순 / 생성순 정렬
- 팝업 / 데스크톱 / 위젯별 기본 정렬 정책 분리
- 사용자 마지막 선택 기억

### 5-4. 완료 항목 자동 정리

- 완료 항목 자동 숨김 또는 아카이브
- n일 지난 완료 항목 정리 규칙
- 완료 탭과 자동 정리 규칙의 관계 정의

### 5-5. 백업 / 내보내기

- `todos.json` 수동 백업
- JSON 내보내기 / 가져오기
- 설정 포함 백업 여부 선택

## 7. 요약

### MVP

“빠르게 추가하고, 오늘 할 일을 보고, 완료할 수 있는가?”

### v1.1

“날짜·태그·우선순위 기준으로 정리할 수 있는가?”

### v1.2

“각 항목을 상세하게 편집하고 관리할 수 있는가?”

### v2

“이 앱만의 차별점과 집중 작업 경험이 충분한가?”

### v3

“장기 사용성과 데이터 관리 편의성이 충분한가?”

가장 중요한 건 처음부터 모든 기능을 넣는 것이 아니라,  
**팝업 + 기본 위젯 + 저장 + 동기화**를 먼저 완성해서 앱의 중심 경험을 만드는 것입니다.
그 다음에 **날짜/태그/우선순위/위젯 상태 저장**을 붙이고, 이후에 **상세 편집**, 마지막에 **데스크톱 뷰/타이머/태그 위젯**, 그리고 **검색/반복/백업** 같은 확장 기능으로 가는 것이 가장 자연스럽습니다.
